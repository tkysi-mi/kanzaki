# Kanzaki 🏯

> あらゆるアウトプットの品質を、自分で定義したルールでチェックする pre-commit ツール

## Kanzaki とは

**Kanzaki** は、git commit 時に変更内容を LLM（大規模言語モデル）で自動レビューする CLI ツールです。

従来のリンターは構文やフォーマットしかチェックできませんでした。Kanzaki は LLM を使うことで、**意味的な品質チェック** を実現します。

### 従来のリンターとの違い

| | 従来のリンター | Kanzaki |
|---|---|---|
| チェック対象 | 構文・フォーマット | 意味・内容・品質 |
| 対象ファイル | コードのみ | コード、ドキュメント、リサーチ、プレゼン、何でも |
| ルール定義 | 専用の設定ファイル | **Markdownのチェックリスト** |
| カスタマイズ性 | ルールセットに依存 | 自然言語で自由に定義 |

### 解決する課題

- 📝 **リサーチ・ドキュメント** — 出典の記載漏れ、事実誤認、用語の不統一を自動検出
- 🎨 **プレゼン・レポート** — トーンの一貫性、構成の完全性をチェック
- 🔒 **チーム運用** — ルールファイルをリポジトリに含めることで、チーム全体の品質基準を統一

```
git commit
  ↓
pre-commit hook が自動起動
  ↓
┌──────────────┐  ┌──────────┐  ┌────────────┐
│ .kanzaki.md  │  │ git diff │  │ ファイル全文 │
│ (ルール定義)  │  │ (staged) │  │ (コンテキスト)│
└──────┬───────┘  └─────┬────┘  └──────┬─────┘
       └────────────┬───┘─────────────┘
                    ↓
           ┌──────────────┐
           │   LLM API    │
           │ OpenAI/Claude │
           └──────┬───────┘
                  ↓
  ✓ [error] ルール1: パス
  ✗ [error] ルール2: 失敗 → コミットをブロック
  ⚠ [warn]  ルール3: 失敗 → 警告のみ（コミットは続行）
```

### check の処理フロー

`kanzaki check` は以下の順序で実行されます：

**① ステージ確認** — `git diff --staged --name-only` で変更ファイルを確認。変更が0件なら何もせず終了。

**② 設定ロード** — APIキーとプロバイダーを以下の優先順位で解決：
  CLIフラグ → 環境変数 → `~/.config/kanzaki/credentials.json`

**③ ルール解析** — `.kanzaki.md` を解析し、3つの要素を抽出：
  - `- [ ]` パターン → ルール（severity + テキスト）
  - `## Header (*.ts)` → グループ名 + ファイルスコープ
  - それ以外のテキスト → LLMへのコンテキスト

**④ Git 情報取得** — `git diff --staged` でdiff全文を取得し、変更されたファイルの全文も読み込む（バイナリや100KB超のファイルはスキップ）。

**⑤ ファイルスコープ フィルタリング** — ルールのglobパターンと変更ファイルをマッチング。関係ないルールを除外することで、トークンを節約し誤検知を防止。

**⑥ プロンプト構築** — 以下の要素を1つのプロンプトに組み立て：

| 部分 | 内容 |
|------|------|
| System Prompt | 「品質レビュアーとして、各ルールへの準拠を判定せよ」 |
| コンテキスト | `.kanzaki.md` 内の自由記述テキスト |
| ルール一覧 | `[ERROR]` / `[WARNING]` タグ付きで列挙 |
| Diff | `git diff --staged` の出力（最大50KB） |
| ファイル全文 | 変更ファイルの完全な内容（各最大20KB） |

**⑦ LLM 呼び出し** — OpenAI は JSON mode、Anthropic はテキスト応答からJSON抽出。各ルールに対して `{ rule, passed, reason }` を返却。

**⑧ 結果出力 + 終了コード**
  - `!error` ルールが1つでも失敗 → **exit(1) でコミットをブロック**
  - `!warn` のみ失敗 → 警告表示して exit(0)（コミットは続行）

## インストール

```bash
npm install -g kanzaki
```

## セットアップ

### 1. 認証

```bash
# APIキーを対話入力（入力はマスク表示 ****）
kanzaki login

# OpenAI サブスクリプションの OAuth を使う場合
kanzaki login --oauth

# Anthropic を使う場合
kanzaki login --provider anthropic
```

認証情報は `~/.config/kanzaki/credentials.json` に保存され、一度ログインすれば全プロジェクトで使えます。

### 2. プロジェクトの初期化

```bash
cd your-project
kanzaki init
```

これにより：
- `.kanzaki.md`（ルールファイル）が作成される
- `.git/hooks/pre-commit` にフックがインストールされる

### 3. ルールのカスタマイズ

`.kanzaki.md` を開いて、プロジェクトに合ったルールを書きます。

### 4. 通常通りコミット

```bash
git add .
git commit -m "update report"
# → Kanzaki が自動でレビューを実行
```

## ルールの書き方

`.kanzaki.md` は Markdown のチェックリスト形式で記述します。

### 基本構文

```markdown
# プロジェクト名のレビュールール

## グループ名
- [ ] ルールの内容（デフォルトで !error 扱い）
- [ ] !error 明示的にエラーとして定義
- [ ] !warn  警告のみ（コミットはブロックしない）
```

### 重要度レベル

| 記法 | 動作 | 用途 |
|------|------|------|
| `!error`（デフォルト） | 違反時にコミットをブロック | 必須ルール |
| `!warn` | 警告表示のみ、コミットは続行 | 推奨ルール |

### 自由記述コンテキスト

チェックリスト項目以外のテキストは、LLM への追加コンテキストとして送信されます。プロジェクトの背景や前提条件を書くことで、LLM の判断精度が向上します。

```markdown
# レビュールール

このプロジェクトは医療機器の規制文書を管理しています。
すべての変更は FDA 21 CFR Part 11 に準拠する必要があります。

## コンプライアンス
- [ ] !error 規制要件への参照が正確であること
- [ ] !error 変更履歴が適切に記録されていること
```

### ファイルスコープ

ヘッダーに括弧で glob パターンを指定すると、そのグループのルールを特定のファイル種別のみに適用できます。

```markdown
## ドキュメント (*.md, *.txt)
- [ ] !error リンク切れがないこと
- [ ] !warn 見出しの階層構造が正しいこと

## コード (*.ts, *.js)
- [ ] !error console.log が本番コードに残っていないこと
- [ ] !error ハードコードされた秘密情報がないこと

## 全ファイル
- [ ] !warn TODO コメントが残っていないこと
```

パターン未指定のグループは全ファイルに適用されます。

## コマンド一覧

| コマンド | 説明 |
|---------|------|
| `kanzaki init` | `.kanzaki.md` 作成 + pre-commit hook インストール |
| `kanzaki check` | ステージ済みの変更をレビュー（hookから自動実行） |
| `kanzaki install` | pre-commit hook のみインストール |
| `kanzaki login` | APIキーまたはOAuthで認証 |
| `kanzaki logout` | 保存済み認証情報を削除 |
| `kanzaki status` | 認証状態を表示 |

### check オプション

```
-p, --provider <名前>   LLMプロバイダー: openai / anthropic（デフォルト: openai）
-m, --model <名前>      使用モデル（デフォルト: gpt-4o）
-r, --rules <パス>      ルールファイルのパス（デフォルト: .kanzaki.md）
    --api-key <キー>    APIキー（kanzaki login 推奨）
    --no-block          警告のみ、コミットをブロックしない
-v, --verbose           詳細な出力を表示
```

## 設定

### 認証方法

| 方法 | コマンド | 適した用途 |
|------|---------|-----------|
| 対話式ログイン | `kanzaki login` | 個人利用（推奨） |
| OAuth | `kanzaki login --oauth` | OpenAIサブスクリプション利用 |
| 環境変数 | `export KANZAKI_API_KEY=sk-...` | CI/CD |

### 環境変数

| 変数 | 説明 | デフォルト |
|------|------|-----------|
| `KANZAKI_API_KEY` | LLM APIキー | — |
| `KANZAKI_PROVIDER` | `openai` / `anthropic` | `openai` |
| `KANZAKI_MODEL` | モデル名 | `gpt-4o` |
| `KANZAKI_RULES_PATH` | ルールファイルのパス | `.kanzaki.md` |

プロジェクトルートの `.env` ファイルは自動的に読み込まれます。
